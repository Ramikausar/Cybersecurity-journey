💠 Tcpdump is a powerful, command-line packet analyzer used to intercept, capture, and log network traffic passing through your computer or server.
💠 The Tcpdump tool and its libpcap library are written in C and C++.
💠 The libpcap library is the foundation for various other networking tools today. Moreover, it was ported to MS Windows as winpcap.

Through this will learn how to:      

        >> Capture packets and save them to a file
        >> Set filters on captured packets
        >> Control how captured packets are displayed

Network Interface :
        >>  A network interface is a connection point between your computer and a network. Each interface can send and receive network traffic.

        Common interfaces include:

                Interface                        Purpose

                eth0, ens5       ->       Wired Ethernet connection
                wlan0	         ->       Wi-Fi connection
                lo	         ->       Loopback interface (localhost traffic)
                any	         ->       All available interfaces

When We run tcpdump, it needs to know where to listen for packets.

If your system has multiple interfaces, traffic may be flowing through only one of them. Capturing on the wrong interface could result in seeing no packets.

To solve this problem we have " -i " (-i INTERFACE)  

steps to select interface for the Tcpdump:
                 
                 1. Finding Available Interfaces :

                                ip address show 
                                     or 
                                ip a s
                                
                    it will list out all the listening interface
                    such as >  eth0, ens5, wlan0, any.

                2.  now add option in command :

                                sudo tcpdump -i <interface name>

                                example 1 : sudo tcpdump -i ens5
                                this will capture all the traffic of ens5 

                                example 2 : sudo tcpdump -i eth0
                                this will capture all the traffic of eth0

Read Captured Packets from a File :

        We can use Tcpdump to read packets from a file by using -r FILE.

        -i  →   listen to live network traffic
        -r  →   replay and inspect saved traffic

Apply Filters While Reading :

        Applying filters while reading is very usefull, we can filter Particular packet in different different way such as port number, ip address, etc. 
        
        example :
                sudo tcpdump -r dns.pcap port 53
                sudo tcpdump -r dns.pcap host 8.8.8.8

Limit the Number of Captured Packets :

        We can specify the number of packets to capture by specifying the count using -c COUNT.
        Without specifying a count, the packet capture will continue till you interrupt it.
            
            syntax :
                sudo tcpdump -i ens5 -c < number to capture packet >

            command :
                sudo tcpdump -i ens5 -c 5

Don’t Resolve IP Addresses and Port Numbers:

        By default, tcpdump tries to make packet information easier to read by converting:

            IP addresses → domain names (DNS lookup)
            Port numbers → service names

        For example, it may replace an IP address with a website name and replace a port number with a service name.

        This name conversion requires extra lookups, which can:

            Slow down packet capturing
            Create extra DNS traffic
            Hide the actual IP addresses and port numbers you need for analysis

        The -n option tells tcpdump: Do not convert IP addresses into domain names. Show the actual IP addresses.

        The -nn option tells tcpdump: Do not convert IP addresses and do not convert port numbers into service names. Show everything in numeric form.

Produce (More) Verbose Output:
        
        Verbose :
                In tcpdump, verbose mode controls how much detail tcpdump prints for each packet.

                -v	->      Displays additional packet details beyond the default output.
                -vv	->      Displays even more protocol-specific information.
                -vvv	->      Displays the maximum amount of detail tcpdump can decode.

        


************************************************** Summary and Examples **************************************************

                Command                 	Explanation

                tcpdump -i              INTERFACE	Captures packets on a specific network interface
                tcpdump -w              FILE	Writes captured packets to a file
                tcpdump -r              FILE	Reads captured packets from a file
                tcpdump -c              COUNT	Captures a specific number of packets
                tcpdump -n              Don’t resolve IP addresses
                tcpdump -nn	        Don’t resolve IP addresses and don’t resolve protocol numbers
                tcpdump -v	        Verbose display; verbosity can be increased with -vv and -vvv
