Filtering by Host :

        Normally, tcpdump captures all network traffic, which can be overwhelming.
        traffic related to only one device or server, we can use :
        host <IP_ADDRESS>, host <HOSTNAME>

        example : sudo tcpdump host example.com -w http.pcap
                  sudo tcpdump host 93.184.216.34

Filtering by source IP :

        We use src when we want to see only packets originating from a specific IP address or hostname.
        we will use ' src ' flag for it 
        
        Syntax  :     tcpdump src host <IP_ADDRESS>
                      tcpdump src host <HOSTNAME>

        Example :
                      sudo tcpdump src host 192.168.1.50

        Captured :
                        192.168.1.50  ---->  Your PC
                        192.168.1.50  ---->  Router
        Not Captured :
                        Your PC       ---->  192.168.1.50
                        Router        ---->  192.168.1.50

Filtering by Protocol :

        The final type of filtering we will cover is filtering by protocol.
        examples include: ip, ip6, udp, tcp, and icmp.

Logical Operators :

        Three logical operators that can be handy:

        and:     Captures packets where both conditions are true. For example, tcpdump host 1.1.1.1 and tcp captures tcp traffic with host 1.1.1.1.
        or:      Captures packets when either one of the conditions is true. For instance, tcpdump udp or icmp captures UDP or ICMP traffic.
        not:     Captures packets when the condition is not true. For example, tcpdump not tcp captures all packets except TCP segments; we expect to find UDP, ICMP, and ARP 
                 packets among the results.
